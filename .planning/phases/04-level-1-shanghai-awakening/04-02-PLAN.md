  <action>
Create `src/scenes/Level1Scene.js`. Delete the old GameScene logic — Level1Scene IS the new game scene.

Key implementation requirements:

1. **World bounds**: `this.physics.world.setBounds(0, 0, LEVEL1.worldWidth, LEVEL1.worldHeight)` — world is 3200×720.

2. **Camera follow**: After creating player, call:
   ```js
   this.cameras.main.setBounds(0, 0, LEVEL1.worldWidth, LEVEL1.worldHeight)
   this.cameras.main.startFollow(this.player.sprite, true, 0.1, 0.1)
   ```
   Lerp 0.1 gives smooth follow without instant lock.

3. **Platform construction** — loop over `LEVEL1.platforms`:
   ```js
   this._platforms = this.physics.add.staticGroup()
   LEVEL1.platforms.forEach(p => {
     const rect = this.add.rectangle(p.x, p.y, p.w, p.h, p.color ?? 0x444466)
     this.physics.add.existing(rect, true)
     // tag moving platforms separately
     if (p.moving) {
       this._setupMovingPlatform(rect, p)
     } else {
       this._platforms.add(rect)
     }
   })
   ```

4. **Ground**: Create from LEVEL1.ground separately (same static body approach).

5. **Moving platform**: Store in `this._movingPlatforms = []`. Use a tween:
   ```js
   _setupMovingPlatform(rect, data) {
     this.physics.add.existing(rect, true)  // static body — manually moved
     const startX = data.x
     this.tweens.add({
       targets: rect,
       x: startX + data.rangeX,
       duration: (data.rangeX / data.speed) * 1000,
       yoyo: true,
       repeat: -1,
       ease: 'Linear',
       onUpdate: () => rect.body.reset(rect.x, rect.y)
     })
     this._movingPlatforms.push(rect)
   }
   ```
   `body.reset()` syncs the static physics body position each frame of the tween.

6. **Player + colliders**:
   ```js
   this.player = new Player(this, LEVEL1.playerSpawn.x, LEVEL1.playerSpawn.y)
   this.player.body.setCollideWorldBounds(true)
   // Collide with ground
   this.physics.add.collider(this.player.sprite, ground)
   // Collide with each static platform rect (can't use staticGroup because moving ones are separate)
   // Add colliders individually for normal platforms
   // For moving platforms:
   this._movingPlatforms.forEach(mp => {
     this.physics.add.collider(this.player.sprite, mp)
   })
   ```
   Actually: collect all non-moving rects in an array and add a single collider call with that array.

7. **Registry + StatsManager**: Mirror the existing GameScene pattern exactly:
   - `initRegistry(this)` is NOT called here — BootScene already seeded registry.
   - `this.stats = new StatsManager()` — load persisted values.
   - `const allStats = this.stats.getAll(); Object.entries(allStats).forEach(([k,v]) => this.registry.set(k, v))`
   - Set checkpoint in registry: `this.registry.set(KEYS.CHECKPOINT_X, LEVEL1.checkpoint.x); this.registry.set(KEYS.CHECKPOINT_Y, LEVEL1.checkpoint.y)`

8. **HUD launch**: `this.scene.launch('HUDScene')`

9. **Shutdown handler** (ARCH-03):
   ```js
   this.events.once(Phaser.Scenes.Events.SHUTDOWN, () => {
     this.scene.stop('HUDScene')
   }, this)
   ```

10. **Death floor**: `this.deathFloorY = LEVEL1.worldHeight + 100`

11. **update()**: call `this.player.update(delta)` + `this.handleDeath()`

12. **respawn()**: `this.player.body.reset(cx, cy)` same as GameScene.

13. **Fade in**: `this.cameras.main.fadeIn(300, 0, 0, 0)`

14. **Scene key**: `super('Level1Scene')` in constructor.

Do NOT add collectibles, enemies, or boss yet — those are Plans 03-05. Leave placeholder arrays:
