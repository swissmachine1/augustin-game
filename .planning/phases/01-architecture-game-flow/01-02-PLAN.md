The Player class wraps a Phaser.GameObjects.Rectangle and attaches its own physics body. Use a factory-style class (not extending Rectangle — Phaser 4 does not support extending game objects the same way as v3). Store the rectangle as this.sprite and the scene reference for later use.

```js
import Phaser from 'phaser'

export class Player {
  constructor(scene, x, y) {
    this.scene = scene

    // Colored rectangle placeholder — swap for sprite in Phase 2
    this.sprite = scene.add.rectangle(x, y, 32, 48, 0x00ff88)
    scene.physics.add.existing(this.sprite)
    this.sprite.body.setCollideWorldBounds(true)

    // Expose body for collision setup in GameScene
    this.body = this.sprite.body
  }

  // Expose position for checkpoint reads
  get x() { return this.sprite.x }
  get y() { return this.sprite.y }

  destroy() {
    this.sprite.destroy()
  }
}
```

Conventions: no semicolons, 2-space indentation, named export only, no default export. Single responsibility: this file does NOT import GameRegistry — that coupling belongs in GameScene.
